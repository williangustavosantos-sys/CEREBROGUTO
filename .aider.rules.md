# Regras obrigatórias para o Aider neste repositório

1. NUNCA deletar funções, classes ou arquivos sem listar EXATAMENTE o que será removido e receber aprovação explícita ("sim", "aprovo", "go").
2. Antes de qualquer edit, mostrar um resumo em português:
   - Arquivo afetado
   - O que será adicionado
   - O que será modificado
   - O que será removido (se algo)
3. Edits devem ser incrementais. Não reescrever arquivos inteiros.
4. Se o prompt for ambíguo, PERGUNTAR antes de agir.
5. Nunca substituir código funcional por comentários, diagramas ou placeholders.
6. Máximo de 50 linhas modificadas por edit sem aprovação prévia.
